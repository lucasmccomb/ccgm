# Multi-Agent System

## Overview

Multiple Claude Code agents work on the same repository in parallel using independent clones.
Each agent runs in its own clone with full git isolation - independent branches, independent PRs.

## Directory Structure

Standard setup for any repo using multi-agent:

```
~/code/{repo}-repos/
├── {repo}-0/           # Clone 0 (agent-0)
├── {repo}-1/           # Clone 1 (agent-1)
├── {repo}-2/           # Clone 2 (agent-2)
├── {repo}-3/           # Clone 3 (agent-3)
└── {repo}-N/           # Additional clones as needed
```

Each clone is a full, independent git repository. Agents work in separate tmux panes, each running Claude Code.

### Setup Commands

```bash
REPO="my-repo"
GITHUB_USER="your-username"
AGENT_COUNT=4  # Adjust as needed

mkdir -p ~/code/${REPO}-repos
for i in $(seq 0 $((AGENT_COUNT - 1))); do
  CLONE_DIR="$HOME/code/${REPO}-repos/${REPO}-${i}"
  git clone git@github.com:${GITHUB_USER}/${REPO}.git "$CLONE_DIR"
  git -C "$CLONE_DIR" checkout -b agent-${i} origin/main
  echo "# Auto-generated by clone setup. Do not edit.\nCLONE_NUMBER=${i}" > "$CLONE_DIR/.env.clone"
done
```

**Important**: Ensure `.env.clone` is in the project's `.gitignore`. If not, add it after setup:
```bash
echo ".env.clone" >> ~/code/${REPO}-repos/${REPO}-0/.gitignore
```

### Agent Labels (create once per repo)

```bash
# Color palette (cycles for agents beyond 7)
AGENT_COLORS=("0075ca" "1d76db" "5319e7" "0e8a16" "d93f0b" "e4e669" "006b75" "b60205")

for i in $(seq 0 $((AGENT_COUNT - 1))); do
  gh label create "agent-${i}" --color "${AGENT_COLORS[$i % ${#AGENT_COLORS[@]}]}" \
    --description "Being worked on by agent-${i}"
done
```

See `~/.claude/github-repo-protocols.md` for the full label set and repo creation guide.

### Adding an Agent to an Existing Repo

To add a new agent (e.g., agent-4) to an already-running repo group:

```bash
REPO="my-repo"
N=4  # New agent number
GITHUB_USER="your-username"
REPOS_DIR="$HOME/code/${REPO}-repos"
CLONE_DIR="${REPOS_DIR}/${REPO}-${N}"

# 1. Clone
git clone "git@github.com:${GITHUB_USER}/${REPO}.git" "$CLONE_DIR"

# 2. Create idle branch
git -C "$CLONE_DIR" checkout -b "agent-${N}" origin/main

# 3. Create GitHub label (run from any existing clone)
AGENT_COLORS=("0075ca" "1d76db" "5319e7" "0e8a16" "d93f0b" "e4e669" "006b75" "b60205")
cd "${REPOS_DIR}/${REPO}-0"
gh label create "agent-${N}" --color "${AGENT_COLORS[$N % ${#AGENT_COLORS[@]}]}" \
  --description "Being worked on by agent-${N}"

# 4. Write clone number
echo "# Auto-generated by clone setup. Do not edit.\nCLONE_NUMBER=${N}" > "$CLONE_DIR/.env.clone"

# 5. Copy env files (if applicable, excluding .env.clone which was just written)
for f in "${REPOS_DIR}/${REPO}-0"/.env*; do
  [ -f "$f" ] && [[ "$(basename "$f")" != ".env.clone" ]] && cp "$f" "$CLONE_DIR/"
done

# 6. Install dependencies
cd "$CLONE_DIR"
[ -f pnpm-lock.yaml ] && pnpm install
[ -f package-lock.json ] && npm install
```

The logging system auto-derives agent identity from the directory suffix - no log config changes needed. The log file (`agent-N.md`) is created automatically on `/startup`.

## Agent Identity

Derived automatically from the working directory name suffix:
- `my-repo-0` or `my-repo` -> agent-0
- `my-repo-1` -> agent-1, etc.

No suffix or suffix `-0` = agent-0. Numeric suffix = that agent number.

## Dev Server Port Allocation

Each clone gets isolated ports to prevent collisions when multiple agents run dev servers simultaneously.

### Convention

Ports are offset by the clone number from the project's base ports:

| Service | Base Port | Clone 0 | Clone 1 | Clone 2 | Clone 3 | Clone N |
|---------|-----------|---------|---------|---------|---------|---------|
| Frontend (Vite) | 5173 | 5173 | 5174 | 5175 | 5176 | 5173+N |
| Backend (Wrangler/API) | 8787 | 8787 | 8788 | 8789 | 8790 | 8787+N |
| Additional services | project-defined | +0 | +1 | +2 | +3 | +N |

### How It Works

1. **Setup writes clone number**: The clone setup script writes a `.env.clone` file containing `CLONE_NUMBER=N`
2. **Build tools read it**: `vite.config.ts`, wrangler dev scripts, and Playwright configs read `CLONE_NUMBER` to compute their port
3. **Agents use it**: When starting dev servers or configuring test URLs, agents use the designated ports

### .env.clone File

Written automatically during clone setup. Git-ignored. Contains:

```
# Auto-generated by clone setup. Do not edit.
CLONE_NUMBER=N
```

### Agent Instructions for Dev Servers

When running dev servers, **always use clone-aware ports**:

```bash
# Detect clone number from .env.clone or directory name
CLONE_N=$(grep -oP 'CLONE_NUMBER=\K\d+' .env.clone 2>/dev/null || echo "${PWD##*-}")

# Frontend dev server
pnpm dev -- --port $((5173 + CLONE_N))

# Worker/API dev server
wrangler dev --port $((8787 + CLONE_N))
```

Or if the project's `vite.config.ts` reads `.env.clone` automatically (preferred), just run `pnpm dev` and ports are handled.

### Playwright / E2E Testing

When running E2E tests, point Playwright at the correct port:

```bash
CLONE_N=$(grep -oP 'CLONE_NUMBER=\K\d+' .env.clone 2>/dev/null || echo "${PWD##*-}")
BASE_URL="http://localhost:$((5173 + CLONE_N))" npx playwright test
```

Projects should configure `playwright.config.ts` to read `CLONE_NUMBER` from `.env.clone` for the `baseURL` and `webServer.port`.

## Claiming Issues (Before Writing Any Code)

1. **Check GitHub labels** - skip if any `agent-*` or `in-progress` label exists:
   ```bash
   gh issue view {number} --json labels --jq '[.labels[].name]'
   ```

2. **Check sibling clone branches** - skip if any clone has a branch starting with `{issue-number}-`:
   ```bash
   for dir in $(find ~/code/{repo}-repos/ -maxdepth 1 -name "{repo}-[0-9]*" -type d 2>/dev/null); do
     echo "$(basename $dir): $(git -C $dir branch --show-current 2>/dev/null)"
   done
   ```

Both checks must be clear before claiming.

3. **Label the issue immediately** (before creating a branch):
   ```bash
   gh issue edit {number} --add-label "in-progress" --add-label "agent-{N}"
   ```

4. **Verify labels applied**:
   ```bash
   gh issue view {number} --json labels --jq '[.labels[].name]'
   ```

Do NOT create a branch or start coding until labels are confirmed.

## When Finishing an Issue

```bash
gh issue edit {number} --remove-label "agent-{N}"
# If closing:
gh issue close {number} --comment "Completed: {summary}"
gh issue edit {number} --remove-label "in-progress"
```

## Git Workflow

- **Branch from origin/main**: `git checkout -b {issue}-{desc} origin/main` (preferred) or `git checkout main && git pull && git checkout -b {issue}-{desc}`
- **After PR merge**, return to idle: `git fetch origin && git checkout agent-{N} && git reset --hard origin/main && git branch -d {old-branch}`
- **Each clone fetches independently** - run `git fetch origin` in each clone as needed.
- **npm install is per-clone** - each has its own `node_modules/`.
- **Env files are per-clone** - `.env` / `.env.local` must be copied individually (gitignored). `.env.clone` (containing `CLONE_NUMBER`) is unique per clone and must NOT be copied between clones.
- **Stashes are per-clone** - fully isolated, no cross-agent interference.

## Session Logs

All agents write to your agent log repo at `{log-repo-name}/{repo-name}/YYYYMMDD/agent-N.md`.
Per-agent files = no merge conflicts. See `~/.claude/log-system.md` for full protocol.

At session start, read other agents' logs in the same date subdirectory for cross-agent awareness.

## Conflict Resolution

If two agents accidentally claim the same issue:
1. The human supervisor resolves the conflict
2. One agent releases the issue (removes their label)
3. Work continues with clear ownership
